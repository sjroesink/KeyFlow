  - Import `usePracticeStore` from `../store/practiceStore`.
  - Read `loopRegion` from `usePracticeStore`.
  - When `loopRegion` is set, render a colored overlay on the progress bar:
    ```html
    <div
      class="absolute top-0 h-full bg-primary/20 border-l-2 border-r-2 border-primary/50 pointer-events-none"
      style={{
        left: `${(loopRegion.start / duration) * 100}%`,
        width: `${((loopRegion.end - loopRegion.start) / duration) * 100}%`
      }}
    />
    ```
  - Make the progress bar container `relative` (add to classname).

  **Loop region creation via shift+drag:**
  - Add `onMouseDown` handler to progress bar: if `e.shiftKey`, start loop selection.
  - Track drag state with `useRef<{ dragging: boolean; startFraction: number }>`.
  - On mousedown with shift: record `startFraction = (e.clientX - rect.left) / rect.width`, set dragging=true.
  - Add `onMouseMove` and `onMouseUp` handlers (attach to window during drag for reliable tracking):
    - onMouseMove: calculate current fraction, render preview (optional — can skip for v1).
    - onMouseUp: calculate end fraction, compute `start = Math.min(startFrac, endFrac) * duration`, `end = Math.max(startFrac, endFrac) * duration`. Call `usePracticeStore.getState().setLoopRegion({ start, end })`. The store validates minimum 1-second duration.
  - Clean up window listeners on unmount.

  **Normal click behavior:** Keep existing click-to-seek. Only enter loop-selection mode when shift is held.

  Add a small hint text below the progress bar when no loop is set: `"Shift+drag to set loop"` in `text-[10px] text-outline/50`.
